RSA SOP

1. Approved Applications Queue

Review only records in Approved Applications and confirm reviewer decision is present.

Use search/date filters to process by priority and avoid stale approvals.

2. Details and Document Validation

Open Details to verify personal, RSA, and property fields before final submission.

Use document viewer and download/merge tools when records need packaging for downstream processing.

3. Final Submit

Use Final Submit only after complete validation; status changes to sent-to-PFA flow.

This action also triggers payment assignment logic, so avoid submitting incomplete files.

4. Finally Submitted Tab

Use Finally Submitted to audit what has been dispatched and by which RSA officer.

Re-check timestamps and ownership when investigating delayed payment updates.
